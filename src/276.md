# Frontend Instructions for MSJPay Project

## Project Structure & Module Organization

- `recipes/` - Main recipe collection organized by service/category
  - `Custom` - Fine-grained control via custom configuration
- **GitHub Actions**: Scheduled execution and CI/CD
- **ctx.auth.getUserIdentity()**: Returns user info or null if not authenticated
- **Dual-LLM architecture** (Claude Sonnet 4.5) - Semantic pairing with noUncheckedIndexedAccess, define interfaces/types for props
- **Own Database Tables**: Each component maintains its own isolated database tables
- **No direct database access**: Actions cannot use `ctx.db` directly
- **When making release ensure that docs are update and version number is being handled before finishing this task.
